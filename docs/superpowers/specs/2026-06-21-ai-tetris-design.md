# AI 대전 테트리스 — 설계 문서

- **날짜**: 2026-06-21
- **형태**: 로컬 서버에서 실행되는 웹앱 (Vite 개발서버 / 프리뷰 서버)
- **핵심**: 플레이어 vs AI 공격형(가비지 시스템) 테트리스, 네온 신스웨이브 3D 그래픽, 풀스펙 연출
- **기술 스택**: Vanilla TypeScript + Three.js + Vite, 오디오는 WebAudio 자체 합성

---

## 1. 목표 및 성공 기준

플레이어와 AI가 서로 가비지 라인을 주고받으며 싸우는 대전 테트리스를 만든다. "화려한 3D"를 신스웨이브 네온 + 후처리(bloom) 연출로 달성하고, 코어 로직은 프레임워크와 무관한 순수 모듈로 분리해 단위 테스트가 가능하게 한다.

**성공 기준**
1. 두 필드(플레이어/AI)가 동시에 보이며 60fps로 동작한다.
2. 라인 클리어 시 가비지가 상대에게 전달되고, 수신 측은 경고 후 상쇄할 수 있다.
3. AI가 난이도 1·2·3으로 달라지며(시작 화면 선택), 가시적으로 말을 움직인다. AI는 순수 자체 스크립트(LLM/네트워크 호출 없음)다.
4. 라인 클리어·가비지·콤보·테트리스에 파티클/플래시/화면 흔들림 등 연출이 붙는다.
5. BGM/SFX가 자체 합성되어 외부 에셋 없이 재생된다.
6. 코어 로직(board, SRS, 라인 클리어, 가비지, 점수, AI 합법 수)이 단위 테스트를 통과한다.

## 2. 비목표 (YAGNI)
- 멀티 라운드(3선승), AI 프로필/보이스, 온라인 멀티플레이어, 랭킹 서버, 백엔드, 모바일 터치는 제외. 향후 추가 가능 후보로만 기록.

## 3. 게임 규칙 (모던 테트리스 가이드라인 서브셋)

- **필드**: 가로 10 × 세로 20(표시). 내부적으로 상단 2행 버퍼를 포함해 10×22로 다룬다.
- **테트로미노**: I, O, T, S, Z, J, L 7종. 종류별 네온 색 할당.
- **랜덤**: 7-bag(한 봉지에 7종이 1개씩, 봉지 내 순서는 무작위).
- **회전**: SRS(Super Rotation System). I/O를 제외한 4×4, I는 4×4/상태별 오프셋, O는 회전 없음. 월킥 킥 테이블(JLSTZ 공통, I 별도)로 벽/바닥에서도 회전 시도.
- **기본 조작**
  - 좌우 이동, 소프트드롭(가속 낙하), 하드드롭(즉시 낙하+고정), CW/CCW 회전, 홀드.
  - 락 딜레이: 500ms(조각이 바닥에 닿은 후). 이동/회전 시 리셋(최대 15회, 또는 바닥에서 2행 이하 이동 시 리셋).
  - 고스트: 낙하 예상 위치를 반투명으로 표시.
  - 홀드: 1칸 보관, 낙하마다 1회 교체 가능(교체 후 즉시 교체 불가).
  - Next: 다음 3개 표시.
- **중력(낙하 속도)**: 레벨=1부터 시작, 10라인 클리어마다 레벨+1. 초당 행 수(SPS)는 가이드라인 공식 `secondsPerRow = (0.8 - (level-1)*0.007)^(level-1)` 사용(하한 클램프). AI 낙하 속도는 난이도에 따라 별도 스케일.
- **점수(표준)**
  - 싱글=100·레벨배, 더블=300, 트리플=500, 테트리스=800.
  - T-Spin(회전으로 발생) 인정: TS=400, TD=800, TT=1200 (레벨배).
  - 백투백(직전이 테트리스/T-Spin이고 이번도 해당) ×1.5.
  - 콤보: 연속 클리어 시 (콤보수)×50·레벨배.
  - 소프트/하드 드롭 보정 가산.
- **종료(탑아웃)**: 조각이 스폰 위치(버퍼 행)에 겹쳐서 스폰할 수 없거나, 고정된 블록이 표시 영역 상단을 넘으면 패배. 단판 승. 리매치 버튼 제공.

## 4. 대전 / 공격(가비지) 시스템

- **가비지**: 회색 블록 1행으로, 좌우 한 칸은 빈 구멍(무작위 열). 아래에서 밀어 올려짐.
- **공격량 테이블(보내는 가비지 행 수)**
  - 싱글=0, 더블=1, 트리플=2, 테트리스=4
  - T-Spin 싱글=2, T-Spin 더블=4, T-Spin 트리플=6
  - 콤보: 최초 클리어 이후부터 (콤보 단계)×1 가산. 한 번에 보내는 총 가비지 상한 = 8행(초과분은 버림).
  - 백투백(테트리스/T-Spin 연속) +1
- **타깃**: 1:1 매치이므로 항상 상대 필드.
- **수신/상쇄(캔슬)**: 들어오는 가비지는 큐에 쌓이고 약 **1000ms 경고(필드 가장자리 경고바 점멸)** 후 다음 조각 고정 직후에 적용. 그 전에 자신이 라인을 클리어하면 보낼 가비지에서 들어오는 가비지를 1:1로 먼저 상쇄(남은 분량만 상대에게 전달). 이 캔슬/경고 창이 공정성과 손맛의 핵심.
- **승패**: 어느 한 쪽이 탑아웃되면 즉시 종료, 상대 승리.

## 5. AI 상대

- **구현 방침(중요)**: LLM이나 네트워크 호출을 일절 하지 않는 **순수 자체 휴리스틱 스크립트**로 동작. 오프라인, 외부 API/의존성 없음. 결정은 모두 로컬 평가 함수 + 탐색으로만 이루어진다.
- **평가 함수(Dellacherie식 휴리스틱, 가중치 튜닝 대상)**
  - `aggregateHeight`(−), `completedLines`(+), `holes`(−), `bumpiness`(−), 필요시 `wellColumn`(홀드/I용 우물).
- **탐색**: 현재 조각(및 홀드 고려)의 모든 회전×열 조합을 시뮬레이션해 점수화 후 최적 배치 선택. 상위 난이도는 다음 조각까지 1~2수 전망(lookahead) 적용.
- **난이도 (시작 화면에서 1·2·3 선택)**
  - **난이도 1 (Easy)**: 탐욕(1수), 결정 간격 ~900ms, 상위 후보 3개 중 30% 무작위, 종종 구멍 남김.
  - **난이도 2 (Normal)**: 탐욕, ~500ms, 10% 노이즈, 거의 최적.
  - **난이도 3 (Hard)**: 1~2수 전망, ~250ms, 최적, 빠른 APM.
  - 선택기는 1·2·3 숫자로 표기(괄호 안 이름은 보조 표시). 게임 중 표시/미션에 그대로 사용.
- **가시적 행동**: 타깃 위치를 계산한 뒤 즉시 텔레포트하지 않고, 결정 간격 동안 회전→횡이동→하드드롭을 눈에 보이게 재생(관전 만족도↑). 경로가 막히면 현실적으로 조정.
- AI는 동일한 `engine` 인터페이스를 쓴다(로직 재사용, 공평한 규칙).

## 6. 비주얼 / 3D (Three.js + EffectComposer)

- **신 scene 베이스**: 어두운 공간, 신스웨이브 그리드 바닥(원근 소멸), 별빛/부유 입자, 포그. 미세한 카메라 시점 변화(idle).
- **필드 렌더**: 두 보드(좌=플레이어, 우=AI)를 약간 기울인 투시로 동시 관찰. 각 셀은 테트로미노별 발광 머티리얼의 둥근모서리 박스(RoundedBoxGeometry)이며 `InstancedMesh`로 1드로우콜화. 활성 조각/고스트는 별도 표현(고스트=반투명 와이어/에지).
- **후처리**: `EffectComposer` + `UnrealBloomPass`(네온 글로우의 핵심) + 옵션(비네트, 미세 크로마틱 어버레이션, 필름 그레인). 픽셀비 상한(~2), `powerPreference:'high-performance'`, 적응형 블룸 해상도.
- **연출 이펙트**
  - 라인 클리어: 해당 행 플래시 → 파편 파티클 파열 → 행 스케일/용해로 소거. 클리어 행 수에 비례한 화면 흔들림.
  - 가비지 수신: 필드 가장자리 경고바 점멸(1초) → 회색 행이 아래에서 충격과 함께 밀려올라옴 + 임팩트 셰이크.
  - 테트리스/콤보/T-Spin: 대형 네온 텍스트 팝업 + 전체 화면 플래시 + 카메라 펀치 줌.
  - 하드드롭: 낙하 잔영(모션 블러 느낌의 잔상) + 착지 먼지.
  - 조각 스폰: 살짝 어둡다가 글로우인.
- **HUD(네온 DOM 오버레이, 캔버스 위에 절대배치)**: 점수, 라인, 레벨, 콤보, Next×3, 홀드, 수신 가비지 미터, 타이머/상태. 텍스트는 DOM이 가장 선명.

## 7. 오디오 (WebAudio 자체 합성)

- **에셋 없이 자급자족**: 외부 음원 다운로드/라이선스 이슈 회피. `AudioContext` 기반으로 SFX와 BGM을 절차적 합성.
- **BGM**: 신스웨이브 풍 베이스/아르페지오 루프(시퀀서). 루핑.
- **SFX**: 이동(짧은 블립), 회전, 하드드롭(임팩트), 라인 클리어(피치업 칭), 테트리스/콤보(화려한 화음), 가비지 수신(로우 임팩트), 홀드, 게임오버.
- **컨트롤**: 음소거 토글 + 마스터 볼륨. (실제 음원 파일을 덧붙이는 옵션은 후순위 훅으로 열어둠.)

## 8. 조작 / 입력

- ←/→(또는 A/D) 이동, ↓(또는 S) 소프트드롭, **Space** 하드드롭, ↑/X 회전 CW, Z(또는 Ctrl) 회전 CCW, C(또는 Shift) 홀드, **Esc/P** 일시정지, **R** 재시작(게임오버 시).
- **DAS 170ms / ARR 50ms**(설정 가능 후보). 키 매핑은 표에 정리.

## 9. 아키텍처 / 코드 구조

```
src/
  core/            # 프레임워크 무관 순수 로직(단위 테스트 대상)
    tetromino.ts   # 모양, 색, SRS 상태/킥 테이블
    board.ts       # 10x22 그리드, 충돌, 라인 클리어, 고정
    randomizer.ts  # 7-bag
    rules.ts       # 중력(레벨→SPS), 점수, 공격 테이블, 락딜레이
    engine.ts      # 1인용 상태머신: 스폰/이동/회전/고정/홀드/가비지 큐/탑아웃
    ai.ts          # 휴리스틱 평가 + 탐색 + 가시적 행동 결정
  versus/
    match.ts       # 두 engine 연결, 가비지 라우팅/상쇄, 승패 판정, 리매치
  render/
    scene.ts       # 조명/포그/그리드 바닥/별빛/카메라/후처리 컴포저
    boardView.ts   # 필드별 InstancedMesh 블록, 활성/고스트 표현
    effects.ts     # 파티클, 플래시, 화면 흔들림, 텍스트 팝업, 펀치 줌
    input.ts       # 키보드 + DAS/ARR
    audio.ts       # WebAudio SFX/BGM 합성
  ui/
    hud.ts         # DOM 네온 HUD
    menu.ts        # 시작/난이도 선택, 일시정지, 게임오버 오버레이
  main.ts          # 부트스트랩, 게임 루프, 상태머신(Menu→Playing→Paused→GameOver)
```

- **게임 루프**: 고정 타임스텝(60Hz 로직) + 누적기(accumulator). `requestAnimationFrame`으로 매 프레임 렌더. 속도가 프레임률과 무관하게 일정.
- **상태 머신**: `Menu → Playing → Paused → GameOver`.
- **의존성 방향**: `render`/`ui` → `core`/`versus` (단방향). 코어는 렌더를 모름 → 순수 단위 테스트 가능.

## 10. 성능

- 블록은 필드당 `InstancedMesh`(드로우콜 최소화). 파티클/이펙트는 풀링. 픽셀비 상한, 블룸 해상도 적응형. 목표: 일반 노트북 60fps.

## 11. 테스트

- **Vitest**로 코어 로직 단위 테스트: 보드 충돌/고정/라인 클리어, SRS 회전+월킥(경계 케이스), 7-bag 분포/공정성, 점수/공격 테이블, 가비지 큐/상쇄, AI가 항상 합법 수를 반환, 탑아웃 판정.
- 렌더/연출은 수동 검증.

## 12. 빌드/실행

- `npm install`
- `npm run dev` → Vite 로컬 개발서버(예: http://localhost:5173)
- `npm run build` + `npm run preview` → 프로덕션 로컬 서버
- TypeScript `strict`.

## 13. 리스크 / 메모

- Bloom 성능: 저사양에서 해상도/임계값 조정으로 대응.
- AI "가시적 행동"이 로직 속도와 충돌하지 않도록 결정→애니메이션 시퀀스를 분리된 단계로 구현.
- WebAudio 자동재생 정책: 최초 사용자 제스처(시작 버튼)에서 AudioContext resume.
- (환경 메모) 현재 작업 머신에 git 미설치 → 설계 문서 커밋은 생략, 파일만 작성.
